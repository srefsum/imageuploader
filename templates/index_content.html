How it works

Upload — Select a destination directory and file. Files are saved with sanitized names to prevent path traversal.

Browse — Gallery pages list files as thumbnails. Click a file to view the full image.

Subdirectories — #replace "subdir_directory_name" supports subfolders. You can create new folders from that gallery page.

Serve — Files are served directly via /serve?directory=<key>&file=<name> with MIME detection and caching.

Operational constraints

  • Configured directories: #replace "directory_list"
  • Maximum upload size: 20 MB per request
  • Supported image extensions: .jpg, .jpeg, .png, .gif, .webp
  • Filenames are sanitized to alphanumeric characters, dots, underscores, and hyphens

Security notice

This service has no authentication enabled. Directory access is restricted by an allowlist of configured paths. Do not expose this server to untrusted networks without adding access controls.
